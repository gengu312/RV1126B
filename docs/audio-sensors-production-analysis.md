# RV1126B 音频、触觉传感器与机器人主控应用分析

- 硬件：正点原子 `ATK-DLRV1126B` 开发板 + `ATK-CLRV1126B` 核心板
- 系统：Linux 6.1 / Buildroot
- 整理日期：2026-07-17
- 用途：记录音频能力、4～5 路压力/抚摸传感器接入方式，以及机器人产品化时的硬件结构

## 1. 结论摘要

1. 开发板可以传入、保存和播放高质量音频文件。当前实机音频驱动支持 16/24/32 位、1～2 声道、8～96 kHz；ES8390 芯片理论上支持 24 位、最高 192 kHz。
2. 当前板载麦克风和小扬声器主要面向语音、提示音和交互，不属于 Hi-Fi 换能器。数字音频规格较高，不代表板载声学效果本身达到高保真。
3. 可以外接四极耳麦麦克风、USB 麦克风、USB 声卡、耳机、有源音箱和小型无源喇叭，但不同接口的电平和接法不能混用。
4. 4～5 路压力/抚摸传感器完全可以接入。现有 40P 只直接引出两路 0～1.8 V 模拟输入，因此多路传感器推荐通过 I2C 外接 ADC 和电容检测芯片。
5. 不同位置、力度和抚摸方向可以在软件中映射为不同声音、音量、灯效或振动反馈。
6. 量产时可以去掉 180 × 140 mm 的教学开发底板，但不能只使用裸核心板；必须设计一块机器人定制载板。核心板本体约 55 × 45 mm。

## 2. 音频文件的传输、存储和播放

### 2.1 当前实机检查结果

| 检查项目 | 实机结果 |
| --- | --- |
| ALSA 声卡 | `rockchip-es8390` |
| 播放设备 | `hw:0,0`，ES8390 HiFi |
| 录音设备 | `hw:0,0`，ES8390 HiFi |
| 音频格式 | `S16_LE`、`S24_LE`、`S32_LE` |
| 声道数 | 1～2 声道 |
| 采样率 | 8～96 kHz |
| 用户数据分区 | `/userdata` 约 23 GB 可用 |
| 已有软件 | FFmpeg、FFplay、GStreamer、SoX、`aplay`、`arecord`、`mpg123` |
| 已确认解码格式 | WAV/PCM、FLAC、AAC、MP3、Vorbis 等 |

高质量文件可以通过以下方式进入开发板：

- 当前 USB/ADB 连接；
- U 盘或 TF 卡；
- 有线网口；
- Wi-Fi 网络传输；
- 后续应用中的文件导入功能。

文件复制和存储属于数字数据传输，只要文件没有被重新有损编码，传输过程不会降低音质。

### 2.2 理论能力、实机能力和当前应用的区别

| 层次 | 能力或限制 |
| --- | --- |
| 音频文件 | 可以保存 24-bit/192 kHz WAV、FLAC 等文件 |
| ES8390 理论能力 | 双声道 ADC/DAC，24 位，8～192 kHz |
| 当前 Linux 驱动 | 双声道最高 24-bit/96 kHz；也接受 16/32 位容器格式 |
| 当前 RV1126B 实验台 | 16-bit、16 kHz、单声道录音，只能回放最近录音 |

当前实验台中的 16 kHz 单声道配置是程序选择，不是音频硬件上限。对应录音参数位于 [`apps/rv1126b_lab/main.cpp`](../apps/rv1126b_lab/main.cpp) 约第 1455 行。

当前开发板实际使用外置 ES8390 完成模拟录放。底板原理图中的 RV1126B 片内模拟音频部分属于可选参考设计，当前整板未装配、未使用，因此评价现机音质时应以“ES8390 + HT6872 + 麦克风/扬声器”链路为准。

24-bit/96 kHz 双声道无压缩 PCM 的数据量约为 2.07 GB/小时。目前 `/userdata` 的可用空间理论上可以保存约 11 小时；FLAC 通常占用更少，但压缩率取决于音频内容。

192 kHz 文件可以存储和解码，但当前 ALSA 声卡最高开放到 96 kHz，实际播放时需要转换到 96 kHz 或更低采样率。

### 2.3 什么决定实际录音和播放质量

录音链路：

```text
声源 → 麦克风 → 模拟前端/增益 → ES8390 ADC → Linux 音频数据
```

播放链路：

```text
音频文件 → 解码/重采样 → ES8390 DAC → HT6872 功放 → 扬声器/耳机
```

因此，质量由以下因素共同决定：

- 原始文件格式和采样参数；
- ES8390 和驱动配置；
- 麦克风、扬声器和功放；
- 模拟增益、供电噪声和 PCB 布线；
- 扬声器声学腔体、安装位置和机器人外壳；
- 软件是否进行了降噪、自动增益、回声消除或重采样。

ES8390 的理论指标较好：ADC 典型信噪比约 101 dBA，DAC 典型信噪比约 110 dBA。但开发板上的板载麦克风频响大约为 100 Hz～10 kHz，板载扬声器约为 23 mm、8 Ω、额定 0.5 W，主要适合语音和提示音。

因此更准确的结论是：**当前开发板具备较高规格的数字音频通路，但板载换能器是语音交互级，不应直接称为 Hi-Fi 播放或录音设备。**

目前只核验了声卡能力和数据手册参数，还没有使用音频分析仪测量整板实际频响、失真和信噪比。

## 3. 外接麦克风和扬声器

### 3.1 外接麦克风

可选方案：

1. **3.5 mm 四极耳麦麦克风**
   - 底板原理图将 `INP_PHONE_MIC` 接入 ES8390；
   - 当前 ALSA 驱动存在 `Headset Mic Jack`、`Headset Mic Switch` 和 `Main Mic Switch`；
   - 需要使用与插座线序兼容的四极耳麦，独立三段麦克风可能需要转接；
   - 目前尚未插入一副真实耳麦验证录音，具体线序、偏置和自动切换仍需实物测试。
2. **USB 麦克风或 USB 声卡**
   - 当前内核已经注册 USB Audio 驱动；
   - 兼容 USB Audio Class 的设备通常会成为第二张 ALSA 声卡；
   - 具体设备仍需实物测试采样率、供电和驱动兼容性。
3. **量产自定义音频前端**
   - 可通过核心板 SAI/I2S/PDM 连接数字麦克风、麦克风阵列或更高规格 Codec；
   - 需要定制载板、设备树和驱动配置。

> XLR 动圈麦克风、48 V 幻象供电电容麦不能直接连接开发板，需要麦克风前置放大器或专业 USB 声卡。

### 3.2 外接扬声器

| 设备 | 推荐接口 | 说明 |
| --- | --- | --- |
| 普通耳机 | 3.5 mm 接口 | 最直接 |
| 有源音箱 | 3.5 mm 接口 | 改善播放质量时优先采用 |
| 小型无源喇叭 | `SPKL`/`SPKR` | 由两颗 HT6872 D 类功放驱动 |
| USB DAC/声卡 | USB Host | 可绕过板载模拟输出链路 |

HT6872 在 5 V、4 Ω 负载下，典型输出约 2.30 W（1% THD+N），板上左右声道各使用一颗。

> `SPKL`/`SPKR` 是 BTL 功放输出，两根线都不是系统地，任何一根都不能接 GND。有源音箱的线路输入不能接到该端子，应使用 3.5 mm 接口。原理图中 `SPKR` 与板载喇叭并联，外接前还要核算总阻抗和功率。

## 4. 4～5 路压力和抚摸传感器接入

### 4.1 芯片能力不等于现有插口数量

RV1126B 芯片内部有 3 组 SARADC，每组最多 8 路、13 位，属于芯片级最大资源。当前 Linux 实机只注册了一个 SARADC IIO 设备，包含 `in_voltage0_raw`～`in_voltage7_raw`。

现有底板的 40P 只把其中两路模拟输入引出到排针，其他通道被板载电位器、按键、耳机检测等功能使用，或没有引到 40P。因此不能把“芯片有 24 路 ADC”理解为“当前开发板有 24 个外接模拟插口”。

### 4.2 40P 中与传感器最相关的引脚

| 40P 物理脚 | 信号 | 电气域/用途 |
| --- | --- | --- |
| 7 | `SARADC0_IN6` | 模拟输入，正常有效范围 0～1.8 V |
| 8 | `SARADC0_IN1_CDS` | 模拟输入，正常有效范围 0～1.8 V |
| 13 | `I2C2_SDA_M0` | 3.3 V I2C 数据，当前对应 `/dev/i2c-2` |
| 15 | `I2C2_SCL_M0` | 3.3 V I2C 时钟，当前对应 `/dev/i2c-2` |
| 23/25/27/29 | SPI1 CS/CLK/MOSI/MISO | 3.3 V；当前没有 `/dev/spidev*`，需设备树启用 |
| 29/31/33/35 | UART2 TX/RX/RTS/CTS | 3.3 V；当前 UART2 未启用 |
| 36 | 3.3 V | 给低功耗传感器小板供电 |
| 40 | 5 V | 电源，不是信号输入 |
| 16/28/37 | GND | 传感器公共地 |
| 1～6 | FSPI/GPIO | 1.8 V，不建议接普通 3.3 V 模块 |

> 直接进入 40P 第 7、8 脚的模拟信号必须限制在 0～1.8 V 正常量程内。不能把 3.3 V 当作正常满量程，更不能输入 5 V、负压或未经保护的压电尖峰。物理脚号已按 `ATK-DLRV1126B V1.0` 底板原理图第 13 页复核；排针从照片上端开始是 40/39，再依次向下递减。

当前 I2C2 总线上已有 4.7 kΩ 上拉和地址为 `0x51` 的板载 RTC。新增器件需要避免地址冲突。

### 4.3 推荐的第一版原型

```text
2～3 路压力传感器 → 模拟前端/外置 ADC ┐
                                     ├→ 40P I2C → RV1126B
2～3 个抚摸区域   → 电容检测控制器     ┘
```

建议器件框架：

- 压力：使用 [TI ADS1115](https://www.ti.com/product/ADS1115)，提供 4 路单端、16 位、最高 860 SPS 的 I2C ADC；
- 电容抚摸：使用 [TI FDC1004](https://www.ti.com/product/FDC1004)，提供 4 路电容检测和主动屏蔽驱动；
- 共同连接：40P 第 13 脚 SDA、第 15 脚 SCL、第 36 脚 3.3 V，以及第 16/28/39 脚中的任意一个 GND。

2～3 路压力加 2～3 路电容抚摸时，一片 ADS1115 加一片 FDC1004 即可覆盖。若 5 路全部是模拟压力信号，应选一片 8 通道 ADC 或使用两片可配置不同地址的 ADC。

### 4.4 必须先确认传感器类型

“压力传感器”或“电子皮肤”并不是统一电气接口。拿到具体型号和数据手册后，才能冻结电路：

- FSR、压阻膜、弯曲传感器：通过分压电路变成电压，再进入 ADC；
- 应变片、称重片：通常是毫伏级差分输出，需要 HX711、NAU7802 或仪表放大器；
- 压电片：可能产生正负高压尖峰，需要电荷/缓冲放大、限流和钳位；
- 电容式电子皮肤：使用电容检测芯片，不能简单当成普通直流电压；
- 已带 I2C/SPI/UART/USB 的数字传感器：按其接口和电平直接连接；
- 5 V 数字输出：必须经过电平转换，不能直接进入 3.3 V 或 1.8 V GPIO。

## 5. 不同力度和位置产生反馈

软件处理流程：

```text
采样 → 基线校准 → 低通滤波/去抖 → 力度归一化 0～1
     → 判断位置、力度、顺序和速度 → 选择声音、音量、灯效或振动
```

可以实现的交互示例：

- 轻按播放轻柔回应音；
- 重按提高音量，或切换成另一段声音；
- 抚摸脸部、背部或手部播放不同内容；
- 多个触摸区域连续触发时，判断从左向右或从上向下的抚摸方向；
- 结合压力层和电容层，同时判断“摸在哪里”和“用了多大力度”。

压力信号是输入，反馈是程序判断后的独立输出：

- 声音：通过 ALSA、ES8390 和功放输出；
- 指示灯：通过 GPIO 或 PWM 控制；
- 振动器、电机：通过 MOSFET、H 桥或专用驱动器控制；
- 可变模拟电压：需要增加外置 DAC，或使用 PWM 加滤波。

> GPIO 只能提供控制信号，不能直接驱动喇叭、电机、振动器、加热片或大功率灯。

## 6. 机器人中的长线和分布式传感器

桌面原型、线长只有几十厘米时，可以把 ADC 和电容检测器放在一块 40P 传感器汇聚小板上。

机器人脸、背、手等位置距离较远，并且附近可能有电机和开关电源噪声，不建议把微弱模拟信号或普通 I2C 线拉遍全身。更合理的量产结构是：

```text
脸部传感器 → 本地小 MCU/ADC ┐
背部传感器 → 本地小 MCU/ADC ├→ CAN/RS-485 → RV1126B 主控
手部传感器 → 本地小 MCU/ADC ┘
```

当前开发底板已经提供两路 CAN FD 和两路 RS-485 收发器；实机也存在 `can0`、`can1`、`ttyS4`、`ttyS5`。CAN 更适合多个身体节点，RS-485 也适合较长距离的可靠串行通信。

CAN 线束应使用双绞线，采用线性总线结构，120 Ω 终端电阻只放在总线两端。每个身体区域应尽量在本地完成模拟采样、滤波和校准，再向主控发送数字结果。

## 7. 量产时核心板和开发底板的关系

### 7.1 核心板尺寸与组成

`ATK-CLRV1126B` 核心板的主要参数：

- PCB 尺寸约 55 × 45 mm，厚约 1.6 mm；
- 两个 120P BTB 座，共 240 个触点，0.5 mm 间距；
- 5 V 供电；
- 板上主要包括 RV1126B、LPDDR4/4X、eMMC 和电源管理芯片。

当前教学开发底板约为 180 × 140 mm。量产时可以删除这块大底板，但核心板官方参考手册明确说明核心板不能独立工作，必须搭配底板或定制载板。

### 7.2 当前大底板提供了什么

| 资源 | 所在位置 |
| --- | --- |
| RV1126B、内存、eMMC、PMIC | 核心板 |
| 12 V 转 5/3.3/1.8 V、电源保护 | 开发底板 |
| ES8390、麦克风、耳机接口、双路功放 | 开发底板 |
| Wi-Fi/蓝牙模块和两根天线接口 | 开发底板 |
| 千兆网 PHY、RJ45 | 开发底板 |
| USB Hub、USB 接口、4G/SIM | 开发底板 |
| CAN/RS-485 收发器 | 开发底板 |
| MIPI 摄像头和显示屏连接器 | 开发底板 |
| TF、RTC、按键、LED、ADC 电位器、40P | 开发底板 |

因此，量产结构不是“核心板加几根飞线”，而是：

```text
核心板 + 按机器人需求裁剪的定制载板 + 各部位传感器/执行器节点
```

### 7.3 定制载板需要保留的内容

根据机器人功能选择保留：

- 核心板 5 V 电源、电源时序、保护和散热；
- RESET、PWRON、MASKROM/USB 烧录、UART0 和测试点；
- 音频 Codec、麦克风接口、功放和扬声器接口；
- Wi-Fi/蓝牙模块、射频走线和天线；
- 屏幕 MIPI DSI 和摄像头 MIPI CSI 连接器；
- CAN/RS-485、I2C、SPI、UART、GPIO 和 ADC 前端；
- 电机、振动器、灯、加热片等负载的专用驱动电路。

核心板的接口数量属于单项最大能力，多个功能可能复用同一组引脚。定制载板冻结原理图以前，应先制作完整 IO 资源表，并同步确定 Linux 设备树和 pinmux 配置，不能假设 SPI、I2C、UART、CAN、SAI、摄像头等资源可以全部同时使用。

未使用的教学接口可以删除，例如多余 USB 接口、4G 模组座、板载电位器、部分按键和完整工业端子。

### 7.4 外设能否分布到机器人不同部位

可以，但不同信号需要不同连接方式：

- 屏幕和摄像头：MIPI 高速差分信号，应使用短 FFC 排线和受控阻抗走线，不能使用长杜邦线；
- Wi-Fi/蓝牙天线：无线模块仍需放在载板上，通过 50 Ω 射频走线和同轴线把天线移到塑料外壳边缘，避开金属、电机和 DC-DC；
- 麦克风：尽量靠近头部声孔，优先使用本地数字麦克风或短差分模拟线；
- 扬声器：功放尽量靠近扬声器，主控传输音频数据和控制信号；
- 皮肤和压力传感器：各部位本地采集，再通过 CAN/RS-485 汇总；
- 电机、振动器和灯：在负载附近设置驱动器，主控只发送控制命令。

### 7.5 小批量和大批量方案

- 原型和小批量：继续使用正点原子核心板，设计一块定制机器人载板，开发速度快、风险较低；
- 大批量且对成本/体积非常敏感：可以考虑把 RV1126B、DDR 和 eMMC 直接做到自研主板，但需要解决多层板、DDR、高速 MIPI、USB3、射频、EMC、热设计和整机认证，难度明显更高。

最终主控总成可以比当前 180 × 140 mm 开发板小很多，但不会只有 55 × 45 mm，因为仍需要载板、电源、连接器、音频、无线和散热空间。

## 8. 推荐实施顺序

1. 给现有触摸实验台增加“导入并播放 24-bit/96 kHz WAV 或 FLAC”功能。
2. 使用板载喇叭、3.5 mm 有源音箱和外接麦克风分别做对比测试。
3. 向导师或供应商取得拟用压力膜、电子皮肤的准确型号、照片和数据手册。
4. 先制作“ADS1115 + 电容检测器 + 40P I2C”的桌面原型，显示各通道原始值、滤波值和力度曲线。
5. 在程序中实现位置、力度和抚摸方向到声音、灯效或振动反馈的映射。
6. 原型验证后再冻结 IO 资源表，设计机器人定制载板和 CAN 传感器节点。
