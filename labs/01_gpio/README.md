# 实验 01：GPIO

## 目标

1. 理解物理针脚、原理图网络名、pinmux 和 Linux GPIO 编号不是同一个概念。
2. 找到一个经原理图确认、没有被驱动占用的 GPIO。
3. 先做输出高低电平，再做按键输入。

## 当前实机状态

- 内核提供 8 个 GPIO 控制器：`gpiochip0`～`gpiochip7`。
- 旧 sysfs GPIO 接口可用：`/sys/class/gpio/export`。
- libgpiod 命令行工具未安装。
- 多条 GPIO 已被音频、电源、复位、USB 供电等功能占用，不能随便挑编号。

## 现在可以安全执行的只读命令

进入开发板：

```powershell
adb shell
```

在板端查看 GPIO 控制器和已占用线：

```sh
ls -l /dev/gpiochip*
cat /sys/kernel/debug/gpio
ls -l /sys/class/gpio
```

## GPIO 切换前的暂停点

先从整板资料中找到底板原理图和排针图，把候选针脚填入 `docs/pin-map.md`。至少确认：

- 物理位置；
- 原理图网络名；
- IO 电平；
- 当前 pinmux；
- 没有被屏幕、摄像头、音频、电源或复位功能占用。

完成这些信息后再写高低电平脚本。当前仓库故意不提供一个可以随意填写编号并直接拉高的脚本，以免误操作电源、复位或接口控制线。

## 后续验收

- 输出：控制一个安全的板载 LED，或带限流电阻的外部 LED，连续切换 100 次。
- 输入：读取一个安全按键，输出稳定 0/1，并记录上拉/下拉和防抖方式。
- 重启：重启开发板后原有屏幕、音频、USB ADB 功能仍正常。
