# 实验 05：MIPI 摄像头

## 当前结论

摄像头型号已经确认是 `ATK-MCIMX415`。完整连接链路应为：

```text
镜头/传感器小板 -> 原有短黑色软连接 -> ATK-MCIMX415 接口板 -> 2x11 排针 -> 底板 MIPI CSI
```

厂家规格书确认 `ATK-DLRV1126B` 有两路 MIPI CSI，均支持 IMX415。实机系统也已经为两路 IMX415 加载驱动；当前日志中的 `Unexpected sensor id(000000)` 表示没有读到传感器，不表示需要先安装新驱动。

## 灰色排线不接大底板

照片中印有 `ATK-MCIMX415`、末端带 22 根金属针的蓝色小板，就是接口板，不需要再找另一块转接板。

- 镜头小板和接口板之间原有的短黑色软连接已经接好，不要动。
- 用户后来插入背面白色座的灰色 FFC，是另一种主板使用的备用 22 针接口。
- `ATK-DLRV1126B` 使用末端 `2x11` 排针直插底板，灰色 FFC 的自由端不接任何地方。
- 灰色 FFC 自由端触点裸露，不能让它碰到主板后通电。需要取下时必须先解锁背面的 ZIF 座，不能硬拽；锁扣看不清时先补拍近照。

在插摄像头前，先补拍断电后的两个近照：

1. 灰色排线当前所插背面白色座的正上方特写，用于确认怎样安全取下。
2. 松开屏幕固定螺丝、轻轻抬起屏幕后，拍底板上 `MIPI DSI -> MIPI CSI -> MIPI CSI -> CIF` 一排接口及丝印。不要拉扯屏幕灰色排线。

## 确认照片后的安装规则

1. 先正常关机，再拔掉 DC12V、USB 调试线和其他供电线，等屏幕及指示灯完全熄灭。
2. 灰色 FFC 不接底板；取下或可靠隔离裸露触点。
3. 用接口板末端的 `2x11` 公排针插入一个标有 `MIPI CSI` 的母座。22 个针脚必须全部对正，不能错一行或偏一列。
4. 不要插入 `MIPI DSI` 屏幕接口、`CIF` 扩展接口或 40PIN 用户 IO。
5. 镜头朝开发板外侧、背离屏幕和核心板。MIPI 摄像头按不支持热插拔处理。

## 接好后的最小验收

重新冷启动后，先不要解压 SDK，也不要刷机。电脑通过 ADB 检查：

```powershell
adb shell "dmesg | grep -i imx415"
adb shell "v4l2-ctl --list-devices"
```

只接一个摄像头时，另一条未使用的 CSI 仍可能报告探测失败，因此不能要求日志里完全没有错误。验收标准是：已连接的那一路能识别出 IMX415、媒体拓扑出现传感器，并且屏幕上的相机 App 能显示实时画面。设备节点编号必须在接好后动态确认，不照搬其他开发板的 `/dev/videoX`。
